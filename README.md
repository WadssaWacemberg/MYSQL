🚀 MySQL: Do Zero ao Relacional (Generation Brasil)
Bem-vindos ao meu repositório de estudos em Banco de Dados MySQL. migrando de tabelas simples para estruturas complexas com relacionamentos de integridade.

📂 Estrutura do Repositório
----------------------------------------------------------------------------------------------------------------------------------------------------------------
📂 Atividades_Quinta
Nesta pasta encontram-se os fundamentos básicos onde iniciei a manipulação de dados individuais:

RH Empresa: Gestão de colaboradores e filtros salariais.

E-commerce: Cadastro de produtos e controle de estoque.

Registro Escolar: Sistema de notas e filtragem de desempenho.
-----------------------------------------------------------------------------------------------------------------------------------------------------------------

📂 PASTA ATIVIDADE 27
Aqui aplico a técnica da Linha Reta, criando bancos de dados relacionais onde tabelas "Mães" (Primary Key) e "Filhos" (Foreign Key) coexistem em perfeita harmonia.

🛠️ Projetos Implementados:

db_generation_game_online: Gerenciamento de classes e personagens com filtros de poder de ataque e defesa.

db_pizzaria_legal: Sistema de categorias e sabores com filtros de preço e tipo (doce/salgada).

db_farmacia_bem_estar: Controle de estoque farmacêutico e filtros por categorias de medicamentos.

db_cidade_das_carnes: Arquitetura para açougue e hortifruti, focada em setores e precificação.

db_construindo_vidas: Loja de materiais de construção com filtros específicos por departamento.

db_curso_da_minha_vida: Plataforma EAD com cruzamento de dados entre áreas de conhecimento e cursos.

-------------------------------------------------------------------------------------------------------

🧠 Conceitos Aplicados

DDL (Data Definition Language): Criação de bancos e tabelas com CREATE.

DML (Data Manipulation Language): Inserção de dados com INSERT.

DQL (Data Query Language): Consultas avançadas com SELECT, WHERE, BETWEEN e LIKE.

Relacionamentos: Implementação de FOREIGN KEY para garantir a integridade referencial.

Joins: Utilização de INNER JOIN para unificar a linhagem de dados das tabelas relacionadas.
